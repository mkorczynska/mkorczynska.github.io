GIT i GitHub

Wprowadzenie

Git to rozproszony system kontroli wersji (VCS – Version Control System). Służy do śledzenia zmian w plikach, umożliwia pracę zespołową, cofanie zmian, tworzenie alternatywnych wersji projektu oraz bezpieczne eksperymentowanie.

  • działa lokalnie – bez internetu,
  • umożliwia cofanie zmian,
  • wspiera pracę zespołową,
  • pozwala tworzyć alternatywne wersje projektu.

Rozproszony oznacza, że nie istnieje jedno główne repozytorium, z którym łączą się współpracownicy. Każdy posiada na swoim komputerze kopię całego repozytorium. Zaletą takiego podejścia jest możliwość pracy offline.

GIT i GitHub

Podstawowe pojęcia

Repozytorium

Repozytorium to katalog projektu wraz z ukrytą strukturą .git, w której Git przechowuje całą historię zmian.

Rodzaje repozytoriów:

  • lokalne
  • zdalne

Commit

Commit to zapis aktualnego stanu projektu w historii repozytorium. Zawiera informacje o autorze, dacie, opisie zmian oraz wskaźnik do poprzedniego commita. Commit nie zapisuje pojedynczego pliku, lecz cały stan projektu w danym momencie - jeżeli pewne pliki się nie zmieniły, GIT przechowuje tylko łącze do ich poprzednich wersji. Każdy commit ma swój unikatowy identyfikator (SHA1)

Working tree

Working area/working directory (katalog roboczy) to pliki, które użytkownik edytuje, obszar, w którym dokonuje zmian - znajdują się tutaj wszystkie zmiany (nie zcomiitowane i nieznajdujące się w staging area).

Staging area

Staging area (index) to przestrzeń pośrednia pomiędzy working area a repozytorium. Trafiają tu zmiany dodane poleceniem git add. Tylko pliki znajdujące się w staging area zostaną zapisane w commicie - polecenie git commit stworzy commit na podstawie plików w tym obszarze.

Branch

Branch (gałąź) to niezależna linia rozwoju projektu. Pozwala pracować nad nową funkcjonalnością bez wpływu na główną wersję projektu. Domyślną gałęzią jest zazwyczaj main. Każda gałąź to oddzielny katalog roboczy i staging area. Jeżeli praca w ramach gałęzi zostanie ukończona, możliwe jest scalenie jej z główną częścią projektu.

Merge

Merge (scalanie) to operacja pozwalająca na połączenie ze sobą gałęzi (branch).

Remote

Remote to zdalne repozytoria Git, najczęściej umieszczone na serwisach takich jak GitHub lub GitLab. Umożliwiają współpracę zespołową oraz synchronizację zmian pomiędzy różnymi komputerami.

Tagowanie

Tagowanie polega na oznaczaniu konkretnych commitów etykietami (tagami), np. wersjami v1.0 lub release-2025. Tagi są najczęściej używane do oznaczania stabilnych wydań projektu.

Aliasy

Aliasy to skrócone nazwy poleceń Git definiowane przez użytkownika. Pozwalają uprościć i przyspieszyć pracę, np. alias git st zamiast git status.

HEAD

HEAD to wskaźnik pokazujący, na którym commicie aktualnie pracuje użytkownik. Zwykle wskazuje na ostatni commit bieżącej gałęzi. Zmienia się z każdym nowym commitem lub podczas przejścia na inny commit poprzez mianę gałęzi (branch).

Konfiguracja

Polecenie git config pozwala na ustawienie opcji konfiguracyjnych takich jak np. nazwa użytkownika czy adres email

Opcje konfiguracyjne przechowywane są w trzech plikach - różnią się one zakresem obowiązywania

  • local <repo> /.git/gonfig - domyślny zakres działania polecenia git config (jeśli nie została podana flaga, domyślnie jest --local). Ustawienia obowiązują dla repozytorium.
  • global /.gitconfig - poziom globalny odnosi się do konkretnego użytkownika systemu operacyjnego (flaga --gllobal)
  • system $(prefix) - poziom systemu obejmuje cały komputer i dotyczy wszystkich jego użytkowników

Kolejność:m local-->global-->system

git config --global user.name "Jan Kowalski"
git config --global user.email "jan@example.com"

Konfiguracja ta jest zapisywana globalnie i używana we wszystkich repozytoriach.

Określa ona nazwę autora oraz adres mailowy dla wszystkich commitów w bieżącym repozytorium.

Flaga --global powoduje, że ustawienia zostaną zastosowane dla bieżącego użytkownika.

git config --system core.editor <editor>

Powyższe polecenie umożliwia ustawienie edytora tekstu dla wszystkich użytkowników urządzenia (flaga --system)

W polu <editor> należy podać nazwę edytora np. vim, atom, emacs, nano.

Kolorowanie danych wyjściowych

Git umożliwia korzystanie z kolorowania składni danych wyjściowych

git config --global color.ui false

powyższe polecenie wyłączy kolorowanie składni. Domyślnie color.ui ustawione jest na auto.

Można zastosować szczegółowe ustawienia kolorów dla konkretnych poleceń

Praca z repozytorium

Tworzenie repozytorium

git init

Polecenie inicjalizuje nowe repozytorium w bieżącym katalogu.

Sprawdzenie statusu

git status

Pokazuje: zmodyfikowane pliki, pliki gotowe do commita, pliki nieśledzone.

Dodawanie plików i commit

git add plik.txt
git commit -m "Pierwszy commit"

Commit zapisuje aktualny stan plików w historii projektu.

Praca na branchach

git branch

lista branchy

git checkout -b nowa-funkcja

Branch pozwala rozwijać nowe funkcje bez ryzyka uszkodzenia głównej wersji projektu.

Mergowanie branchy

git checkout main
git merge nowa-funkcja

Aliasy

git config --global alias.st status
git config --global alias.cm "commit -m"

Tagowanie wersji

git tag -a v1.0 -m "Pierwsza wersja"
git push --tags
